Relative, absolute, and fixed postioning

Ever wondered how websites create those fancy elements that stick to the side as you scroll, or pop-up boxes that stay put? That's the magic of CSS positioning! In this post, we'll explore three powerful tools: relative, absolute, and fixed positioning, and shed light on when to use each one.

Imagine your webpage as a blank canvas. By default, elements like text and images flow from top to bottom, one after another. Positioning allows us to break free from this flow and arrange things exactly how we want.



1. Relative Positioning: Shifting Elements around

Think of relative positioning as gently nudging an element a bit. It stays in its original spot in the document flow, but we can move it up, down, left, or right using the top, right, bottom, and left properties. Relative allows you to actually change the postion of the element relative to where it normally would be in the document flow if it was statically positioned (fyi - static position is the default).

2. Absolute Positioning: Taking Elements Out of Flow

Absolute positioning is more drastic. It completely removes the element from the normal document flow, like taking it out of the puzzle entirely. Fixed positioning is like using super glue for elements. They stay in a fixed spot in the viewport (the area you see in your browser window) regardless of scrolling. This is perfect for navigation bars or "back to top" buttons that you want to be visible at all times.
